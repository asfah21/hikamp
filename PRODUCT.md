# Product

## Register

product

## Users

Mosque administrators and IT staff who manage audio broadcast systems across multiple Hikvision devices. They are technically literate but not necessarily developers — they need clear, efficient tools to manage devices, schedules, audio files, and prayer time broadcasts. They work in a mosque/religious institution context and need reliability above all.

## Product Purpose

Hikamp is a centralized management dashboard for Hikvision audio broadcasting systems. It allows administrators to manage devices, upload and sync audio files, schedule broadcasts, configure prayer time calculations and broadcasts, and monitor broadcast logs — all from a single web interface. Success means administrators can reliably manage audio broadcasts across dozens of devices without touching each device individually.

## Brand Personality

Reliable, precise, calm. The product should feel like professional infrastructure — trustworthy and capable, not flashy or trendy. Think "industrial-grade tool with a clean interface."

## Anti-references

- Overly gamified dashboards with badges, progress rings, and achievement-style UI
- Dark, "hacker" aesthetic dashboards
- Cluttered CRUD panels with dense tables and no visual hierarchy
- Generic Bootstrap-style admin templates

## Design Principles

1. **Clarity over density** — Every screen should make the primary task obvious. Data density is earned, not default.
2. **Trust through precision** — Alignment, spacing, and typography communicate reliability. Sloppy UI undermines confidence in the system.
3. **Progressive disclosure** — Show what's needed, hide what's not. Complex features (prayer config, device sync) should reveal complexity gradually.
4. **Consistency is safety** — Repeating patterns (buttons, tables, forms, badges) reduce cognitive load. Operators under time pressure should never guess where controls are.
5. **Respect the operator** — The interface should be as professional as the equipment it manages. No unnecessary decoration, no cuteness.

## Accessibility & Inclusion

- WCAG 2.1 AA minimum
- Full keyboard navigation for all management tasks
- Dark mode for low-light environments (server rooms, early morning operations)
- Clear error states with actionable messages
